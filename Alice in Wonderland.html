Finding related characters in Lewis Carrolls' "Alice in Wonderland"¶

The purpose of this notebook is to show how to use the Gensim library (http://radimrehurek.com/gensim/) to find related characters and keywords appearing in text. The workflow is inspired from Sujit Pal's blog (http://sujitpal.blogspot.cz/2014/08/topic-modeling-with-gensim-over-past.html).

So let's begin by importing some libraries:

Now let's create a dictionary. This is simply the vocabulary from the book, adding an integer id to each unique word. We will remove words appearing only once, as well as words appearing more than a certain threshold, which is set after getting some feedback.

In [5]: 
dictionary = gensim.corpora.Dictionary(parse(line) for line in open(INPUT_FILE))
THRESHOLD = 0.6
stop_ids = [dictionary.token2id[stopword] for stopword in stopwords if stopword in dictionary.token2id]
once_ids = [tokenid for tokenid,docfreq in dictionary.dfs.iteritems() if docfreq ==1]
freq_ids = [tokenid for tokenid,docfreq in dictionary.dfs.iteritems() if docfreq >= THRESHOLD*len(dictionary)]
dictionary.filter_tokens(stop_ids+once_ids)
dictionary.compactify()

Once the dictionary is set, we create the corpus. The corpus is a collection of tuples (id,score), where id is the integer id of the word created on the dictionary in the previous step, and score is the frequency of the word.

In [6]: 
corpus = [dictionary.doc2bow(parse(line)) for line in open(INPUT_FILE)]

Now we create the tfidf representation of the corpus. The tf-idf score of each word tells us how important that word is on the given line with respect to the rest of the book.

In [7]: 
tfidf = gensim.models.TfidfModel(corpus, normalize=True)
corpus_tfidf = tfidf[corpus]

So what's going on? We represent the book as a huge (but sparse) matrix where each row is a line of the book and the columns are the words (the dictionary), and on each non-zero entry we write down the tf-idf score.

What we want is to decompose this matrix in a smaller subspace, reducing collinear rows. This is achieved by singular values (LSI, latent semantic indexing).

We have no idea of how many classes of characters-context (topics) should there be. But we can look at the two dimensional projection, and look at the total inertia as a function of the number of clusters. The inertia is the sum of square distances from every point to its nearest centroid.

In [8]: 
max_k = 10
ks = range(1,max_k+1)

lsi = gensim.models.LsiModel(corpus_tfidf,id2word=dictionary, num_topics = 2)
X=[]
for vector in lsi[corpus]:
    if len(vector)!= 2:
        continue
    X.append(np.array([vector[0][1],vector[1][1]]))
X = np.array(X)

# Calculate the third derivative of the inertia

inertias = np.zeros(max_k)
diff = np.zeros(max_k)
diff2 = np.zeros(max_k)
diff3 = np.zeros(max_k)
for k in ks:
    kmeans = KMeans(k).fit(X)
    inertias[k - 1] = kmeans.inertia_
    # first difference    
    if k > 1:
        diff[k - 1] = inertias[k - 1] - inertias[k - 2]
    # second difference
    if k > 2:
        diff2[k - 1] = diff[k - 1] - diff[k - 2]
    # third difference
    if k > 3:
        diff3[k - 1] = diff2[k - 1] - diff2[k - 2]

elbow = np.argmin(diff3[3:]) + 3

We compute the third "derivative", to get the inflection points. The edge of the elbow is then the minimum.

Finally, we run Latent Dirichlet Allocation. This model assumes that each line in the book is generated as a mixture of topics, from which the number is known, and tries to get back the topics that generated them. In our case, a topic corresponds to certain contexts on which the characters appeared, or certain events happened (for instance, the tea party, the trial, etc.) So the topics are the "main events" of the book, somehow.

In []: 
NUM_TOPICS = ks[elbow]

lda = gensim.models.LdaModel(corpus, id2word=dictionary, num_topics=NUM_TOPICS, passes = 10)
lda.print_topics(NUM_TOPICS,num_words=20)
